## Структура на файловете

```
/
├── extract_sign_dictionary.py   # скрипт за извличане от PDF
├── sign_dictionary.html         # уеб интерфейс за търсене
├── index.json                   # генерира се при извличането
├── index.sqlite                 # генерира се при извличането
├── images/                      # генерира се при извличането
│   ├── 00001_АКАДЕМИЯ_АКАДЕМИЧЕН.jpg
│   ├── 00002_АРЕСТ.jpg
│   └── ...
└── problems.log                 # генерира се при извличането
```

## Стъпка 1 — Инсталиране на зависимости

Необходим е Python 3.10 или по-нов.

```bash
pip install pymupdf pillow
```

## Стъпка 2 — Извличане от PDF

Поставете PDF файла в същата папка като скрипта, след което:

```bash
# Пълно извличане (препоръчително)
python extract_sign_dictionary.py "Речник на българския жестов език.pdf"
```

Скриптът автоматично открива кои страници съдържат знаци и пропуска уводните страници, таблиците и индексите.

След приключване ще видите:

```
Auto-detected content pages: 8 – 647 (640 pages)
Extracted 4100 entries
JSON  → index.json
SQLite → index.sqlite
All entries have numbers. Looks clean!
```

### Опции

```bash
# Тест на конкретни страници (препоръчително преди пълно извличане)
python extract_sign_dictionary.py речник.pdf --pages 2-10

# С подробен изход за диагностика
python extract_sign_dictionary.py речник.pdf --pages 2-5 --debug
```

### Изход

| Файл / папка | Съдържание |
|---|---|
| `images/` | По едно JPG изображение на знак, именувано `00001_АКАДЕМИЯ.jpg` |
| `index.json` | Пълен индекс с номер, етикет, файл и страница за всеки знак |
| `index.sqlite` | SQLite база данни за директно търсене |
| `problems.log` | Страници или знаци с проблеми при извличането |

### Търсене в базата данни

```bash
sqlite3 index.sqlite "SELECT * FROM signs WHERE label LIKE '%МАЙКА%'"
sqlite3 index.sqlite "SELECT COUNT(*) FROM signs"
```

## Стъпка 3 — Стартиране на уеб интерфейса

Браузърът не позволява зареждане на локални файлове директно (`file://`), затова е необходим локален сървър. Използвайте един от следните начини:

### Python (най-лесно)

```bash
# Влезте в папката с файловете
cd /path/to/folder

# Стартирайте сървъра
python3 -m http.server 8000
```

След това отворете в браузъра:

```
http://localhost:8000/sign_dictionary.html
```

Сървърът работи докато терминалът е отворен. Спира се с `Ctrl+C`.

### Node.js

```bash
npx serve .
```

### VS Code

Инсталирайте разширението **Live Server**, натиснете десен бутон върху `sign_dictionary.html` → *Open with Live Server*.

## Използване на уеб интерфейса

- **Въведете изречение** на български в полето — знаците се показват автоматично докато пишете
- **Кликнете върху снимка** за уголемен изглед (затваря се с `Escape` или клик)
- **Варианти** — ако за дадена дума има повече от един знак, появяват се номерирани бутони за превключване
- **Споделяне** — появява се линк с изречението, който може да се копира и изпрати

### Как работи търсенето

| Случай | Поведение |
|---|---|
| Точно съвпадение | Намира директно |
| Думи до 3 букви (с, и, в…) | Само точно съвпадение |
| Думи 4–6 букви | Допуска 1 буква разлика |
| Думи 7+ букви | Допуска 2 букви разлика |
| Пунктуация (,.!?) | Автоматично се премахва |
| Ненамерена дума | Показва се с `?` placeholder |

## Бележки

- Изображенията са извлечени директно от PDF без преоразмеряване — качеството е същото като в оригинала
- Знаци с повече от едно изображение (многофазови движения) се записват като едно наредено изображение
- `problems.log` съдържа редове за знаци без намерен надпис — обичайно са декоративни елементи
- Уеб интерфейсът работи изцяло офлайн след първото зареждане на `index.json`
